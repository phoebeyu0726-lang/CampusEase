# AGENTS.md

## Project Overview

本專案是「幸福通勤導航（Happy Commute Navigator）」的前端展示 Demo。

目標是做出一個可以在瀏覽器中直接開啟的靜態網站，展示：

- 校園與周邊通勤路網
- YouBike 站點資訊
- 天氣（降雨）
- 高溫曝曬區域
- 空氣品質（AQI）
- 無障礙設施位置
- 最佳幸福通勤路線推薦
- 不同需求下的智慧路徑規劃

本專案主要用於課堂展示與競賽提案，不是實際導航系統。

## 驗收標準（重要）

### 1. 基本執行

- 使用者可以直接打開 `index.html`
- 不需要安裝額外套件
- 不需要登入
- 不需要後端伺服器
- 不需要 API Key
- 頁面載入後不能出現錯誤畫面
- Console 不應有影響功能的 JavaScript Error

---

### 2. Demo 流程

使用者必須能完成一個完整展示流程：

#### Demo Path A（通勤大學生）

1. 進入首頁
2. 選擇宿舍 → 教室
3. 顯示天氣、AQI、YouBike資訊
4. 選擇「最舒適路線」
5. 系統產生推薦通勤方案
6. 顯示幸福指數與路線結果

#### Demo Path B（無障礙導航）

1. 選擇起點與目的地
2. 啟用「無障礙模式」
3. 系統避開樓梯與陡坡
4. 導航至自動門與電梯
5. 顯示無障礙幸福路線

#### Demo Path C（外地旅客）

1. 選擇景點或校園建築
2. 顯示當前天氣與AQI
3. 顯示YouBike租借資訊
4. 推薦最佳移動方式
5. 顯示完整路線結果

整個流程須能在 3 分鐘內完成展示。

---

### 3. 核心功能

至少完成 MVP 中定義的核心功能。

核心功能不能只是靜態文字說明，必須有：

- 可操作按鈕
- 可切換模式
- 可顯示結果
- 可展示資料結構效果

主要按鈕需有明確用途：

- 計算路線
- 切換導航模式
- 顯示資料結構
- 查看路線資訊

不可存在看似重要但沒有功能的按鈕。
